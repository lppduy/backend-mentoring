# 🎯 PHẦN 2: TASK THIẾT KẾ

## 📌 Context

Thiết kế API cho hệ thống quản lý order đơn giản.

Frontend cần:

* Lấy danh sách orders (có pagination)
* Lấy chi tiết 1 order
* Tạo order mới
* Huỷ order

---

## 📌 Yêu cầu

Đối với mỗi endpoint, phải thiết kế:

1. HTTP method
2. URL
3. Query parameters (nếu có)
4. Request body (nếu có)
5. Response body
6. Status code
7. Ít nhất 2 error case

---

## 📌 Ví dụ mẫu (domain: Quản lý tướng trong quân đoàn Tam Quốc)

### Lấy danh sách tướng trong quân đoàn

- **Method:** `GET`
- **URL:** `/api/legions/leg_shu/generals`
- **Query params:** `?page=1&limit=10&role=warrior`
- **Request body:** không có
- **Response body:**

```json
{
  "data": [
    {
      "id": "gen_01",
      "name": "Triệu Vân",
      "role": "warrior",
      "kingdom": "shu",
      "power": 97,
      "status": "active"
    },
    {
      "id": "gen_02",
      "name": "Gia Cát Lượng",
      "role": "strategist",
      "kingdom": "shu",
      "power": 100,
      "status": "active"
    }
  ],
  "page": 1,
  "limit": 10,
  "total": 35
}
```

- **Status code:** `200 OK`
- **Error cases:**
  1. `400 Bad Request` — `role` không hợp lệ (phải là warrior, strategist, archer, cavalry)
  2. `404 Not Found` — `legion_id` không tồn tại
